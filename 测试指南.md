## 启动步骤

### 步骤 1: 进入项目目录
```bash
cd E:\myProject\FabricEye\backend
```

### 步骤 2: 创建虚拟环境
```bash
# Windows
python -m venv venv

# 激活虚拟环境
venv\Scripts\activate
```

### 步骤 3: 安装依赖
```bash
pip install -r requirements.txt
```

**依赖列表：**
- fastapi - Web框架
- uvicorn[standard] - ASGI服务器
- sqlalchemy - ORM
- aiosqlite - 异步SQLite
- opencv-python - 视频处理
- numpy - 数值计算
- aiohttp - 异步HTTP客户端
- python-multipart - 文件上传
- pydantic - 数据验证
- pydantic-settings - 配置管理
- python-jose - JWT认证
- passlib - 密码哈希

### 步骤 4: 启动服务
```bash
# 开发模式（自动重载）
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 生产模式
uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
```

### 步骤 5: 验证启动
打开浏览器访问：
- API文档: http://localhost:8000/docs
- 健康检查: http://localhost:8000/health
- OpenAPI规范: http://localhost:8000/openapi.json

## 常见问题解决

### 问题 1: ModuleNotFoundError
**症状：**
```
ModuleNotFoundError: No module named 'app'
```

**解决：**
确保在项目根目录 (backend/) 下运行命令，并激活虚拟环境。

### 问题 2: 端口被占用
**症状：**
```
Address already in use
```

**解决：**
```bash
# 查找占用8000端口的进程
netstat -ano | findstr :8000

# 或使用其他端口
uvicorn app.main:app --reload --port 8080
```

### 问题 3: OpenCV安装失败
**症状：**
```
ERROR: Could not build wheels for opencv-python
```

**解决：**
```bash
# 升级pip
pip install --upgrade pip setuptools wheel

# 重新安装
pip install opencv-python
```

### 问题 4: SQLite权限错误
**症状：**
```
PermissionError: unable to open database file
```

**解决：**
确保 data/ 目录存在且有写入权限：
```bash
mkdir -p data
chmod 755 data  # Linux/Mac
```

## Docker测试（可选）

如果本地已安装Docker，可以使用容器测试：

### 方法 1: 使用 Docker Compose
```bash
cd E:\myProject\FabricEye

# 启动服务
docker-compose -f docker-compose.test.yml up --build

# 在另一个终端测试
curl http://localhost:8000/health

# 停止服务
docker-compose -f docker-compose.test.yml down
```

### 方法 2: 手动构建镜像
```bash
cd E:\myProject\FabricEye\backend

# 构建镜像
docker build -t fabriceye-backend .

# 运行容器
docker run -d \
  -p 8000:8000 \
  -v $(pwd)/data:/app/data \
  -v $(pwd)/storage:/app/storage \
  -e DATABASE_URL=sqlite+aiosqlite:///./data/fabric_eye.db \
  --name fabriceye-test \
  fabriceye-backend

# 测试
curl http://localhost:8000/health

# 停止并删除
docker stop fabriceye-test
docker rm fabriceye-test
```
