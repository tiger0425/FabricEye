## 1. 项目概述

### 1.1 项目背景

中国纺织行业年营收超4万亿元，面料质检是产业链关键环节。传统人工验布依赖熟练工人，存在效率低（20-30米/分钟）、漏检率高（15-20%）、人力成本上升（6-10万/年/人）等痛点。

国内机器视觉市场2025年达210亿元，纺织AI质检细分市场约5-10亿元，年增长率20%。但现有方案针对大型纺织企业，单价30-100万元，小微面料商（年营收500万-5000万）无力承担。

### 1.2 项目目标

开发一套**面向小微面料商（经销商、小型加工厂）**的低成本AI验布系统：

- **初期投入**：≤8,000元（传统方案1/5-1/10）
- **月运营成本**：≤500元
- **检测能力**：支持10-15类常见面料缺陷
- **处理模式**：边录边分析，单卷布一个视频
- **技术路线**：前期纯API调用，零训练成本

### 1.3 核心价值主张

| 价值维度 | 传统方案 | 本方案 |
|---------|---------|--------|
| 初期投入 | 30-100万元 | 6,300-7,980元 |
| 部署周期 | 3-6个月 | 1-3天 |
| 技术门槛 | 需专业团队 | 即插即用 |
| 缺陷类别 | 固定3-5类 | 可扩展10-15类 |
| 运维成本 | 年费5-15万 | 月费300-500元 |

## 2. 用户画像与场景

### 2.1 目标用户

#### 用户类型A：小微面料经销商
- **规模**：5-20人，年营收500-3000万
- **痛点**：客户投诉率高（5-8%），退货损失大
- **需求**：出货前快速质检，生成检测报告
- **预算**：一次性投入<1万，月费<500

#### 用户类型B：小型面料加工厂
- **规模**：20-50人，年营收1000-5000万
- **痛点**：成品检验人力不足，交付延期
- **需求**：提升检验效率，降低次品流出
- **预算**：一次性投入<2万，可接受分成模式

#### 用户类型C：面料贸易商
- **规模**：10-30人，多品种小批量经营
- **痛点**：面料种类多，缺陷标准不统一
- **需求**：灵活配置检测标准，支持多品类
- **预算**：轻资产模式优先，偏好SaaS订阅

### 2.2 核心使用场景

#### 场景1：出货前质检（高频）
```
用户：面料经销商李老板
场景：一批5000米涤纶布即将出货给服装厂
流程：
1. 将布卷装上普通验布机
2. 启动系统开始录制视频

4. 生成质检报告，含缺陷位置、类型、截图

```

1. 抽样检验（每卷布抽检10%）
2. 录制样本视频
3. 系统自动识别潜在缺陷
4. 不合格品拒收或协商降价
预期：减少原料损失10-15%
```

#### 场景3：库存盘点（低频）

1. 批量录制库存布卷视频
2. 系统分类统计缺陷类型
3. 生成库存质量报告

预期：库存周转率提升20%
```

# 双路处理架构示意
class StreamingProcessor:
    """
    双线程架构：
    - 录制线程：持续写入视频文件
    - 分析线程：读取已写入的帧，调用API分析
    """
    
    def __init__(self):
        self.video_queue = Queue()      # 帧队列
        self.result_queue = Queue()     # 结果队列
        self.recording_thread = None    # 录制线程
        self.analysis_thread = None     # 分析线程
    
    def start_recording(self, roll_id):
        """启动录制线程"""
        self.recording_thread = Thread(
            target=self._record_video,
            args=(roll_id,)
        )
        self.recording_thread.start()
    
    def start_analysis(self, roll_id):
        """启动分析线程"""
        self.analysis_thread = Thread(
            target=self._analyze_stream,
            args=(roll_id,)
        )
        self.analysis_thread.start()
```

# API请求示例（DeepSeek-VL）
import requests

def detect_defect(frame_path):
    """调用VLM API检测缺陷"""
    
    with open(frame_path, 'rb') as f:
        image_data = base64.b64encode(f.read()).decode()
    
    prompt = """
    请分析这张布料图像，识别以下缺陷类型：
    1. 破洞 (Hole)
    2. 污渍 (Stain)  
    3. 色差 (Color Variance)
    4. 断经 (Warp Break)
    5. 断纬 (Weft Break)
    6. 异物 (Foreign Matter)
    7. 皱褶 (Wrinkle)
    8. 纬斜 (Skew)
    9. 稀密路 (Thick & Thin)
    10. 棉结 (Nep)
    
    请以JSON格式返回：
    {
        "has_defect": true/false,
        "defects": [
            {
                "type": "缺陷类型",
                "type_en": "英文名称",
                "confidence": 0-100,
                "severity": "轻微/中等/严重",
                "bbox": [x1, y1, x2, y2]
            }
        ]
    }
    """
    
    response = requests.post(
        "https://api.deepseek.com/v1/chat/completions",
        headers={"Authorization": f"Bearer {API_KEY}"},
        json={
            "model": "deepseek-vl",
            "messages": [
                {
                    "role": "user",
                    "content": [
                        {"type": "text", "text": prompt},
                        {"type": "image_url", "image_url": f"data:image/jpeg;base64,{image_data}"}
                    ]
                }
            ]
        }
    )
    
    return parse_result(response.json())
```

| 布卷长度 | 500米 | 标准布卷 |
| 检验速度 | 20米/分钟 | 人工验布速度 |
| 检验时长 | 25分钟 | 500÷20 |
| 帧提取频率 | 1帧/秒 | 平衡成本与精度 |
| 总帧数 | 1,500帧 | 25×60×1 |
| 单帧成本 | ¥0.015 | DeepSeek-VL平均 |
| **单卷检测成本** | **¥22.5** | 1,500×0.015 |

#### 4.4.2 月度成本估算

| 用户规模 | 月检验卷数 | 月检测成本 | 基础服务费 | 总计 |
|---------|-----------|-----------|-----------|------|
| 小型商户 | 20卷 | ¥450 | ¥299 | **¥749** |
| 中型商户 | 50卷 | ¥1,125 | ¥299 | **¥1,424** |
| 大型商户 | 100卷 | ¥2,250 | ¥299 | **¥2,549** |

> 注：基础服务费包含系统维护、云存储、技术支持

#### 4.4.3 成本优化策略

1. **帧率自适应**：无缺陷区域降低帧率，有缺陷区域提高帧率
2. **智能采样**：相邻帧相似度高时跳过分析
3. **批量折扣**：预充值API费用享受阶梯折扣
4. **模型优化**：3个月后迁移到本地YOLO模型，成本降至1/10
