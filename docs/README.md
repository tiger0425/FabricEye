## ⚠️ 核心要求

### 开始任何开发前，必须：

1. **阅读并理解完整规范**
2. **在快速参考卡上签字确认**
3. **遵守10步开发流程**
4. **每步完成后获得用户确认**

### 绝对禁止（红线）：

- ❌ 文件超过限制行数
- ❌ 不写测试就提交功能
- ❌ 直接修改已有接口
- ❌ 复制粘贴代码
- ❌ 未获确认就修改核心代码

## 📋 开发流程速查

```
步骤1: 输出设计文档
        ↓
步骤2: 用户确认 ✅
        ↓
步骤3: 写接口定义
        ↓
步骤4: 用户确认 ✅
        ↓
步骤5: 写测试（红）
        ↓
步骤6: 用户确认 ✅
        ↓
步骤7: 实现代码（绿）
        ↓
步骤8: 用户确认 ✅
        ↓
步骤9: 重构优化
        ↓
步骤10: 完成 ✅
```

**每步完成后必须说：**
> "第X步完成，输出：...\n请检查并确认是否继续？"

## 📏 文件大小限制

| 文件类型 | 最大行数 | 超过时处理 |
|---------|---------|----------|
| Python文件 | 150行 | 拆分模块 |
| Vue文件 | 200行 | 拆分为组件+composables |
| TypeScript工具 | 100行 | 按功能拆分 |
| API路由 | 100行 | 按资源拆分 |

**检查命令：**
```bash
# Python文件
find backend/app -name "*.py" -exec wc -l {} + | awk '$1 > 150 {print $1, $2}'

# Vue文件
find frontend/src -name "*.vue" -exec wc -l {} + | awk '$1 > 200 {print $1, $2}'
```

## 🆘 如果出现问题

### 发现代码问题

1. **立即停止开发**
2. **回滚修改**
   ```bash
   git diff --name-only
   git checkout HEAD -- <file-path>
   ```
3. **分析报告原因**
4. **按规范重新开发**

### 联系项目负责人

如果发现规范有遗漏或不清晰的地方：
- 在本文档中添加注释
- 通知项目负责人更新规范
- 等待规范更新后再继续
